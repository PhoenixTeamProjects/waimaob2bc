## 1. 项目性质

- 网站域名：`waimaob2bc.com`，继续使用。
- 网站类型：以中文内容为主的 Phoenix 个人品牌、实战博客与课程服务网站。
- 网站不是产品型 B2B 企业站，不建设产品中心、产品分类或产品详情系统。
- 总纲源自旧 B2B 外贸网站规划；只复用个人品牌、博客内容、课程服务、媒体证明、SEO、咨询和后台数据架构中适用的部分，不继承产品展示与产品获客逻辑。
- 网站需要基于后续提供的项目开发专属后台，核心管理文章、栏目、专题、标签、项目案例、媒体资料、信任数据、课程、咨询和网站设置。
- 已确认部署模式：GitHub + Hostinger + VPS。GitHub 是源码主线；Hostinger 绑定仓库分支后自动拉取代码，根据根目录 `package.json` 安装依赖、执行构建并发布前台；VPS 默认用于后台、数据库、备份、日志、反代或辅助服务，不默认作为前台主站发布入口。

## 4.1 最高优先级隐私红线

- 网站、代码仓库、公开文档和可索引技术输出中，不得出现用户已声明需要隐藏的私人业务方向、产品名称、项目名称、域名、客户开发、供应链、代工、经营情况或案例细节。
- 不得通过文章、案例、标签、分类、搜索结果、SEO 元数据、Schema、媒体库文件名、图片 Alt、站内搜索索引、Sitemap、后台演示数据、代码注释、README 或部署文档间接泄露。
- 历史资料中若仍存在私人业务相关残留文字，全部视为应删除的历史残留，不能作为网站内容、演示数据或仓库公开文档来源。
- 后续收到任何资料时，若其中再次出现相关内容，默认按私人业务信息处理，除非用户以后明确书面解除这一限制。

## 5. 内容方法与表达风格

- 内容优先回答：做了什么、为什么这样做、遇到什么问题、如何解决、结果如何、怎样复盘。
- 培训方法：方法 → 工具 → 操作 → 执行 → 结果 → 复盘。
- 语言风格：真实、直接、有观点、有过程、有实战证据。
- 少用“赋能、聚势、链接全球、重塑增长、商业闭环、全球化新生态”等空泛词语。
- 禁止成功学、焦虑营销、虚假案例、收入承诺、获客承诺和成交保证。
- 截图与案例必须隐藏客户隐私、邮箱、电话、商业敏感信息、API Key、Token、密码及登录凭证。

## 7. 首页与页面原则

- 首页是个人品牌首页，不是简单的博客文章列表。
- 用户进入后 5–10 秒内应知道 Phoenix 是谁、做什么、有什么不同。
- 首页围绕 Hero、真实数据、个人介绍、核心方向、职业经历、培训成果、真实项目、最新内容、课程培训、媒体报道和最终 CTA 展开。
- Hero 核心表达：Phoenix 不是站在外贸之外讲外贸，而是自己每天就在做客户开发、阿里国际站、独立站、海外社媒、AI 和 SOHO。
- Hero 必须使用 Phoenix 本人真实高质量照片，不使用 AI 虚构商务人物。
- “关于我”用品牌故事表达，不做成传统求职简历。
- “联系我”需明确适合咨询的问题，并支持课程、企业内训、一对一咨询、商务合作等咨询类型。

## 7.1 内容与案例系统

- 文章不是“标题 + 日期 + 正文”的简单博客；需支持栏目、专题、标签、内容类型、摘要、封面、作者、阅读时间、SEO、相关文章、相关专题与相关案例。
- 内容类型由后台管理，初始可包括实战记录、方法教程、工具实测、项目复盘、观点、案例和踩坑记录。
- 编辑器需支持 H2–H4、段落、加粗、斜体、链接、列表、引用、图片、图库、表格、代码块、Callout、信息/警告框、步骤、视频与分隔线。
- 项目案例使用独立数据模型，不完全以普通文章代替；未完成项目应如实标记“进行中”，不得包装虚假成果。
- 课程 CTA 应克制且与文章语境相关，禁止高频插入和过度销售。

## 8. 视觉原则

- 方向：专业、清晰、现代、真实、有个人感。
- 推荐：白色与浅灰背景、深色正文、清晰品牌主色、Phoenix 真人照片、真实工作和培训照片、项目截图及媒体资料。

所有页面都要回答：

> 为什么一个正在做外贸、经营企业或者研究 AI 的人应该关注 Phoenix？

## 11. 后台与数据架构

- 核心模块至少包括：`posts`、`categories`、`topics`、`tags`、`authors`、`projects`、`courses`、`media_coverage`、`trust_metrics`、`contact_submissions`、`navigation`、`site_settings`。
- 禁止把全部内容塞入一个万能 `content` 表。
- 前后台从第一天解耦：前端负责展示、SEO、交互与内容消费；后台负责内容、分类、标签、项目、课程、媒体、信任数据、咨询和网站设置。
- 前端通过 Service / Repository / Data Adapter 层访问数据，例如 `getPosts()`、`getPostBySlug()`、`getProjects()`、`getCourses()`、`getMediaCoverage()` 和 `getTrustMetrics()`。
- 前端不得严重耦合某个 CMS 的私有字段，以便未来替换为 Phoenix 自研后台。
- 后台应考虑模块化、权限、中国团队使用习惯、多站点、多语言及后续扩展。
- 网站设置可管理 Logo、Favicon、Phoenix 头像、默认 SEO、OG 图片、邮箱、微信二维码、社交账号、页脚及分析验证信息。
- API Key、Token、服务器凭证等敏感信息只能使用安全环境变量或 Secret 管理，不能存入普通公开数据库字段。
